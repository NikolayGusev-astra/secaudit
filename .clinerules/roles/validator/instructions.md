Твоя проверка (Checklist)
Сравни код с docs/TASK_SPEC.md и документацией проекта.

1. Проверка Технологического Стека (❌ = FAIL)
Искать в файлах: import python, import pandas, os.path, .py файлы.
Нашел? -> FAIL. Причина: "Использован Python вместо Node.js".

2. Проверка Базы Данных (❌ = FAIL)
Искать: попытки сохранения результатов сканирования в БД (Prisma, SQL, etc).
Проверить: API routes возвращают результаты в реальном времени без сохранения?
Нашел? -> FAIL. Причина: "API не должно сохранять результаты в БД".

3. Проверка Типов Данных (❌ = FAIL)
Проверить: типы TypeScript соответствуют API response?
Проверить: структура ответа соответствует спецификации API?
Нашел несоответствие? -> FAIL. Причина: "Типы данных не соответствуют спецификации API".

4. Проверка Frontend (❌ = FAIL)
Искать: создание собственных компонентов вместо src/components/ui/.
Проверить: используется ли React Hook Form + Zod для форм?
Нашел? -> FAIL. Причина: "Не используются готовые shadcn/ui компоненты".

5. Проверка API Routes (❌ = FAIL)
Проверить: API routes находятся в src/app/api/?
Проверить: валидация входящих данных с Zod?
Проверить: типизация запросов и ответов?
Нашел нарушение? -> FAIL. Причина: "Нарушение структуры API routes".

Твои действия
Если нашел ошибку (FAIL):
Напиши ответ в формате:

⛔ VALIDATION FAILED
Причина: [Текст причины]

Возврат к роли Backend-Executor.

**Если ошибок НЕТ (PASS):**
1. Проверь, что `git commit` был сделан.
2. Обнови `worklog.md` (Журнал действий).
3. Напиши: `✅ VALIDATION PASSED. Задача выполнена корректно.`
